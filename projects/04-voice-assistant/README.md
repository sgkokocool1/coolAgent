# 项目 04：语音助手状态机脚手架

## 项目目标

把语音 Agent 从“脚本顺序执行”提升为“状态机驱动”：显式管理空闲、聆听、思考、播报和被打断状态。

## 运行

```bash
python3 projects/04-voice-assistant/main.py
```

## 状态说明

- **空闲**：等待唤醒词或按钮。
- **聆听**：采集音频并交给 VAD。
- **思考**：ASR 文本进入 Agent，必要时调用工具。
- **播报**：TTS 分块输出，支持低延迟首包。
- **被打断**：用户插话时停止当前 TTS，并把新语音送回聆听/思考链路。

## 任务说明

1. 阅读 `VoiceAssistantStateMachine.transition()`，理解所有状态变化都应可观测。
2. 将固定 `demo_audio_events()` 替换成来自麦克风或 WebSocket 的事件。
3. 在 `INTERRUPTED` 状态中增加策略：丢弃旧回复、保存旧回复或合并上下文。
4. 为状态机增加超时：长时间无语音回到空闲，ASR 超时提示重试。
5. 把状态变化输出到日志或前端，用于调试用户体验。

## 验收标准

- 运行脚本后能看到状态迁移日志。
- 打断发生后，旧播报会停止，新意图继续处理。
- 新增状态时，不破坏既有 Mock 流水线。
