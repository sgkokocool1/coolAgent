# 第 17 章：TTS、流式播报与低延迟体验

TTS（Text To Speech）决定语音 Agent “说出来”的体验。用户不会感知你的 LLM token 多漂亮，但会立刻感知声音是否自然、首包是否慢、能不能打断、数字是否读错。

## 1. TTS 在链路中的位置

```text
LLM -> TTS -> Speaker
```

TTS 接收文本，输出音频流。语音 Agent 中的 TTS 不应等待完整回复再开始，而应尽量做到：

- LLM 生成第一句后立即合成。
- TTS 首包尽快播放。
- 后续文本分块排队。
- 用户打断时取消未播放内容。

教学示例中的 `MockTTS` 用分块打印模拟流式音频，第一轮会在播报中途停止。

## 2. CosyVoice

CosyVoice 是常见中文语音合成方案之一，关注自然度、音色克隆和多语言能力。适合：

- 中文语音助手。
- 需要定制音色的产品。
- 希望兼顾自然度和可控性的服务。

工程关注点：

- 推理延迟：首包时间是否满足交互。
- 音色授权：克隆音色需要合规。
- 部署资源：GPU/CPU、并发和队列。
- 文本规范化：数字、日期、英文读法。

## 3. Fish Speech

Fish Speech 偏向高自然度和音色表达，适合对声音表现要求较高的场景：

- 虚拟角色。
- 长文本朗读。
- 更丰富的语气表达。

需要注意：

- 高自然度模型通常推理成本更高。
- 实时流式能力要单独验证。
- 角色音色要保持一致，避免同一 session 中声音漂移。

## 4. GPT-SoVITS

GPT-SoVITS 常用于少样本音色克隆和社区创作场景。优点是可玩性强、音色定制灵活。适合：

- 原型验证。
- 个性化声音。
- 私有部署实验。

生产使用时要特别关注：

- 音色来源授权。
- 输出稳定性。
- 推理并发。
- 对长文本的韵律控制。

## 5. Edge TTS

Edge TTS 接入简单，适合快速 demo 和低成本原型：

- 不需要自己部署模型。
- 多语言、多音色可选。
- 调用方式简单。

限制：

- 依赖外部服务和网络。
- 可控性有限。
- 企业合规和稳定性需要评估。

如果只是教学或内部原型，Edge TTS 很方便；如果是企业核心系统，应准备可控的服务端 TTS 或降级方案。

## 6. 流式 TTS

流式 TTS 的目标是降低“用户说完到听到声音”的时间。常见流水线：

```text
LLM token -> 句子缓存 -> TTS chunk -> 播放队列
```

关键点：

- 按句号、逗号、语义短句切块。
- 第一块尽量短，但不能短到语义不完整。
- 后续块可以并行合成。
- 播放队列要支持取消。

例如：

```text
LLM 输出：杭州今天小雨转多云，22 到 27 度，建议出门带伞。
块 1：杭州今天小雨转多云。
块 2：22 到 27 度。
块 3：建议出门带伞。
```

## 7. 情感和风格

TTS 不只是读字。不同场景需要不同风格：

- 客服：稳定、礼貌、语速中等。
- 陪伴：更自然、有情绪。
- 导航：短促、清晰、低歧义。
- 企业审批：正式、克制。

情感控制可以通过：

- SSML 或模型控制标签。
- 提示词描述风格。
- 不同音色。
- 标点和句长控制。

不要滥用情感。天气提醒可以轻松，权限拒绝和安全提示应保持严肃。

## 8. 文本规范化

LLM 输出的文本不一定适合直接合成。TTS 前建议做规范化：

- `2026/07/17` -> `2026 年 7 月 17 日`
- `￥5000` -> `五千元`
- `API` -> `A P I` 或“接口”
- `3.5%` -> `百分之三点五`
- 删除 Markdown 表格、代码块和链接噪声。

语音回复要短。长列表可以先播报摘要，再问用户是否继续。

## 9. 打断与取消

TTS 必须支持取消：

1. 播放器停止当前音频。
2. 清空未播放队列。
3. 通知 session 当前回复未完成。
4. 新用户语音进入 ASR/Agent。

如果 TTS 服务本身不能取消，也要至少取消本地播放和忽略迟到音频。否则用户会听到旧回复和新回复重叠。

## 10. 延迟预算

一个可用的语音 Agent 可以先设定目标：

| 阶段 | 目标 |
| --- | --- |
| VAD 端点后 ASR final | 200-800 ms |
| LLM 首 token | 300-1000 ms |
| TTS 首包 | 200-700 ms |
| 总首响 | 1-2 秒内 |

不同业务可以放宽，但要持续测量。用户通常能接受复杂任务耗时，但不能接受系统没有反馈。

## 11. 本仓库 Mock TTS

`shared/voice_mocks.py` 的 `MockTTS.speak()` 接收文本并按固定长度切块打印：

```text
[MockTTS] >>> 我查到天气了
[MockTTS] >>> ：杭州小雨转
[MockTTS] 用户打断：停止当前播报
```

它没有真实音频，但保留了两个关键行为：

- 分块输出，模拟流式。
- 取消播报，模拟打断。

替换真实 TTS 时，建议保持类似接口：

```python
class TTS:
    def speak(self, text: str, cancel_token) -> None:
        ...
```

## 12. 验收清单

一个语音 Agent 的 TTS 层至少要满足：

- 首包延迟可测量。
- 播放队列可取消。
- 支持短句分块。
- 数字、日期、金额读法正确。
- 工具错误和安全提示有独立话术。
- 播报期间用户可打断。
- TTS 失败时能降级到文本或简短提示音。
